## Create Model Instances

Use `createModel` to instantiate domain entities. It auto-generates `id` and `createdAt`, then validates with the schema.

**Incorrect:**

```typescript
import { v4 } from 'uuid';

// Manually constructing — no validation, base fields scattered
const record = {
  id: v4(),
  createdAt: new Date(),
  sequence: 1,
  name: 'John',
  sn: '12345678',
};
```

Why it fails: No schema validation at creation time. Base field generation is duplicated everywhere. Invalid data silently passes.

**Correct:**

```typescript
import { createModel } from '@domain/model/Base';
import { basicRecordSchema } from '@domain/model/BasicRecord';

const record = createModel(basicRecordSchema, {
  sequence: 1,
  name: 'John',
  sn: '12345678',
  xrayCode: 100,
  division: 'engineering',
  jumin: '900101-1234567',
});
// Result: { id: "<uuid>", createdAt: <Date>, sequence: 1, ... }
```

The `createModel` function:

```typescript
export const createModel = <T extends ZodType>(
  schema: T,
  value: Omit<z.infer<T>, keyof BaseModel>,
) => {
  return schema.parse(value);
};
```

- `id` and `createdAt` are omitted from input — generated by schema defaults
- `schema.parse()` validates and throws `ZodError` on invalid data

### Bulk creation from external data

```typescript
const records = rows.map(row =>
  createModel(basicRecordSchema, {
    sequence: Number(row.sequence),
    name: String(row.name),
    sn: String(row.sn),
    xrayCode: Number(row.xrayCode),
    division: String(row.division),
    jumin: String(row.jumin),
  })
);
```

### Validation error handling

```typescript
try {
  const record = createModel(basicRecordSchema, invalidData);
} catch (error) {
  if (error instanceof z.ZodError) {
    console.log('Validation failed:', error.errors);
  }
}
```
